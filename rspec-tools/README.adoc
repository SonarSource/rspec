rspec-tools: Tools automating RSPEC workflows
=============================================

Install
-------

.Install Pipenv
[source,shell]
----
# macos
$ brew install pipenv
----

.Install rspec-tools
[source,shell]
----
$ cd rspec-tools
$ pipenv install -e .
----


Usage
-----

.See all commands
[source,sh]
----
$ pipenv run rspec-tools --help
----


Development
-----------

.Install dev dependencies

[source,sh]
----
$ pipenv install --dev
----

.Run tests
[source,sh]
----
$ pipenv run pytest
----

.Format
[source,sh]
----
$ pipenv run black rspec_tools tests
$ pipenv run usort format rspec_tools tests
----

.Lint

Note, this step is not run in CI because it is not configured in CAYC mode.
You can also run flake8 in your IDE.
[source,sh]
----
$ pipenv run flake8
----

.Fixtures

When writing a new test, create a fixture rule in `tests/resources/{invalid_rules, rules}`.

In order to generate an HTML file from the ASCIIdoc, you can use [asciidoctor](https://docs.asciidoctor.org/) as follows:

[source,sh]
----
$ asciidoctor rule.adoc
----

.Link-rot detection

ci/validate_links.sh invokes the `rspec-tools check-links` facility which expects HTML files generated based on the rules directory.
The HTML files should be located in similar paths as the original **/rule.adoc files reflecting the rule id and the language specialization.

For example, the original rule structure is as follows:

[source]
```
../rules/S119
├── comments-and-links.adoc
├── description.adoc
├── java
│   ├── metadata.json
│   └── rule.adoc
├── message.adoc
├── metadata.json
├── swift
│   ├── metadata.json
│   └── rule.adoc
└── vbnet
    ├── metadata.json
    └── rule.adoc
```

Then the HTML files will be generated thus:

```
../out/S119
├── java
│   ├── metadata.json
│   └── rule.html
├── metadata.json
├── swift
│   ├── metadata.json
│   └── rule.html
└── vbnet
    ├── metadata.json
    └── rule.html
```
