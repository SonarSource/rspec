== Why is this an issue?

The ``++print++`` statement was removed in Python 3.0. The built-in function should be used instead.


=== Noncompliant code example

[source,python]
----
print '1'  # Noncompliant
----


=== Compliant solution

[source,python]
----
print('1') 
----
