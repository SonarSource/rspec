== Why is this an issue?

Implicit casts discard information when the resulting type has a lower precision than the original type.


=== Noncompliant code example

[source,cpp]
----
int a = 2.1f; // Noncompliant

long double f();
double d = 0;
d += f(); // Noncompliant
----
