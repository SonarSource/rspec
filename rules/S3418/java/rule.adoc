== Why is this an issue?

This rule raises an issue when required properties are not included in a project's pom.


=== Compliant solution

With a properties value of: division, division/name

[source,java]
----
<project //...>
  <division>
    <name>Manufacturing</name>
  </division>

  <!-- ... -->
</project>
----
