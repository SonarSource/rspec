== Why is this an issue?

Java 22 introduced unnamed variable pattern, `_`.
This feature removes the need to name variables that are syntactically required but otherwise unused.
Moreover, it clearly indicates the intent not to use the variable.

To further minimize code clutter, unnamed variables should use the `var` pattern rather than explicit type declarations.
Removing unnecessary information is done to make code more readable, but it brings an additional benefit in maintainability: should the type change in future, the code can remain the same.

== How to fix it
Replace the local variable type with `var`.

=== Code examples

==== Noncompliant code example

[source,java,diff-id=1,diff-type=noncompliant]
----
for (String _ : myIterable) { // Noncompliant
  // ...
}


try (Resource _ = new Resource()) { // Noncompliant
  // ...
}
----

==== Compliant solution

[source,java,diff-id=1,diff-type=compliant]
----
for (var _ : myIterable) {
  // ....
}

try (var _ = new Resource()) {
  // ...
}
----
