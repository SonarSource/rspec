Description

Starting with Java 18 there is a possibility to write code snippets in javadoc comments. Previously the construction <pre>{@code …​ }</pre> was used for this. While you still can use the old approach, snippets give you more possibilities, like easy highlighting, replacing by substring or regex, extracting to a file, linking, etc.

Noncompliant Code Example

/**
 * The code prints "Hello, World!"
 * <pre>{@code                                 // Noncompliant
 *   System.out.println("Hello, World!");
 * }</pre>
 */
public static void main(String... args) {
   ...
}

Compliant Solution

/**
 * The code prints "Hello, World!"
 * {@snippet :
 *  public static void main(String... args) {
 *      System.out.println("Hello, World!");
 *  }
 * }
 */
public static void main(String... args) {
   ...
}
