Description

Why is this an issue?

Path.of() provides a modern and consistent approach to creating Path instances, aligning with factory methods such as List.of() and Set.of(). Additionally, Paths.get() may be deprecated in future versions of Java.

How to fix it

Replace Paths.get() with Path.of().

Code examples

Noncompliant code example

Path path = Paths.get("my/file.txt"); // non compliant

Compliant solution

Path path = Path.of("my/file.txt"); // compliant
