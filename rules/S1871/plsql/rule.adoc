== Why is this an issue?

Having two branches in an ``++IF++``/``++ELSIF++`` chain with the same implementation is at best duplicate code, and at worst a coding error.

[source,sql,diff-id=1,diff-type=noncompliant]
----
IF param = 1 THEN
    sort_order := 0;
    column := 'LastName';
ELSIF param = 2 THEN
    sort_order := 0;
    column := 'LastName'; -- Noncompliant
ELSE
    sort_order := 1;
    column := 'FirstName';
END IF;
----


[source,sql,diff-id=1,diff-type=compliant]
----
IF param = 1 OR param = 2 THEN
    sort_order := 0;
    column := 'LastName';
ELSE
    sort_order := 1;
    column := 'FirstName';
END IF;
----
