Description

In-stream procedures should always have a name.

Why is this an issue?

In JCL, it is expected for a PROC statement inside of a job stream to have a name.

How to fix it

Code examples

Noncompliant code example

//MYJOB   JOB DXXX
//        PROC
//STEP1   EXEC PGM=MYPGM
//        PEND

Compliant solution

//MYJOB   JOB DXXX
//TRPOC   PROC
//STEP1   EXEC PGM=MYPGM
//        PEND
