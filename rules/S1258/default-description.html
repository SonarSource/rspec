The private variables of a class without a constructor will be left uninitialized after object instantiation. In particular, it is wrong to assume that they are initialized to zero. A constructor should be added to initialize the private primitive, pointer, and array members of an object.

Noncompliant Code Example

class Account /* Non-Compliant - should have a constructor to initialize "balance" */
{
public:
  void credit(int amount)
  {
    balance = balance + amount;
  }

private:
  int balance; /* This variable is left uninitialized, and could have any value */
};

int main(int argc, char* argv[])
{
  Account* acc = new Account;
  acc->credit(100); /* The balance is not guaranteed to be 100, it could be really anything */
  return 0;
}

Compliant Solution

class Account /* Compliant */
{
public:
  Account()
  {
    balance = 0; /* Ensures the balance is initialized with zero */
  }

  void credit(int amount)
  {
    balance = balance + amount;
  }

private:
  int balance;
};

/* ... */
