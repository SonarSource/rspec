Description

As stated in the documentation, the methods Objects.isNull and Objects.nonNull only exist to be used as Predicate. When a predicate is not needed, a simple null check should be used, resulting in more concise and readable code.

This rule reports an issue when one of the two methods is not used as a Predicate.

Noncompliant Code Example

if (Objects.isNull(object))

if (Objects.nonNull(object))

Compliant Solution

if (object == null)

if (object != null)

list.stream().filter(Object::nonNull)
