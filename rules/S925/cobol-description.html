Description

Why is this an issue?

Most COBOL environments do not support recursive PERFORM calls, since they can cause unpredictable results. This rule raises an issue when recursive PERFORM calls are used.

Noncompliant code example

PARAGRAPH1.
  PERFORM PARAGRAPH2.

PARAGRAPH2.
  PERFORM PARAGRAPH3.

PARAGRAPH3.
  PERFORM PARAGRAPH1.

Compliant solution

PARAGRAPH1.
  PERFORM PARAGRAPH2.

PARAGRAPH2.
  PERFORM PARAGRAPH3.

PARAGRAPH3.
  DISPLAY "THIS IS PARAGRAPH3".

  if (num > 0) {
    num = num * fact2(x - 1);  // Noncompliant; fact2 calls this function
  }

}

  if (num > 0) {
    num = num * fact(num - 1);  // Noncompliant; fact calls this function
  }
  return num
