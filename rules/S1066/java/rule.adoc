== Why is this an issue?

include::../description.adoc[]

Code like

[source,java]
----
if (condition1) {
  if (condition2) {             // NonCompliant
    /* ... */
  }
}
----

Will be more readable as

[source,java]
----
if (condition1 && condition2) { // Compliant
  /* ... */
}
----

If merging the conditions seems to result in a more complex code, extracting the condition or part of it in a function or variable is a better approach to fix readability:

include::../noncompliant.adoc[]

Can be simplified to

[source,java]
----
if (file != null && isFileOrDirectory(file)) { // Compliant
  /* ... */
}

private static boolean isFileOrDirectory(File file) {
  return file.isFile() || file.isDirectory();
}
----
