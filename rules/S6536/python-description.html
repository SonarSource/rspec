Description

Most letters and characters simply match themselves. However, there are some characters called metacharacters, that donâ€™t match themselves. Instead, they indicate that some pattern should be matched, or they repeat or change parts of the regular expression.

Why is this an issue?

To use these characters as what they match themselves you can use metacharacters escaping, but it makes it harder to read and understand the regular expression.

How to fix it

Instead of escaping it is better to wrap those characters in character class.

Note: not all the metacharacters could be wrapped in character class.

List of metacharacters could be wrapped:

. $ * + ? { } \ | ( )

List of metacharacters couldn’t be wrapped without escaping

[ ] ^

Code examples

Noncompliant code example

import re
pattern = re.compile("\d+\.\d+", re.DOTALL)

Compliant solution

import re
pattern = re.compile("\d+[.]\d+", re.DOTALL)
