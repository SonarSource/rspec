== Why is this an issue?

Signal handling contains implementation-defined and undefined behaviour.


=== Noncompliant code example

[source,cpp]
----
#include <csignal> // Noncompliant 
void my_handler ( int32_t );
void f1 ( ) 
{ 
  signal ( 1, my_handler ); // Noncompliant 
}
----
